# Architecture Decision Records (ADRs)

Use short, focused ADRs to capture decisions that affect architecture, data, conventions, or providers.

- File name: `adr_####-short-slug.md` (zero-padded integer).
- Create/update an ADR per PR when decisions are made.
- Link ADRs in PR body and vice versa.

Numbering:

- Start at `0001` and increment.
- Do not renumber; mark superseded ADRs with status and links.
