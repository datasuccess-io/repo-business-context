# UX Principles

These principles apply to both the framework itself (how users adopt RBC) and any content or artifacts it produces.

## 1. Under 5 minutes to first value

The Quick onboarding promise is the framework's lead. Every interaction that blocks first value has to justify itself.

## 2. No ceremony

Skip options everywhere. "Just go" is always a valid answer. If a user says skip, we write something generic and move on — we never block on a missing answer.

## 3. Autonomous, not gated

The agent writes and updates context files without waiting for approval. Users correct what's wrong in conversation — the same way they correct code. No review ceremonies, no promotion steps.

## 4. Transparent changes

Users always know when context files are touched. The agent announces every write or update in one line. No hidden state, no silent mutations.

## 5. Living over complete

A half-populated `/context/` with current content beats a fully-populated one with stale content. The framework optimizes for freshness, not coverage.

## 6. One answer, many files

The skill routes a single user answer to whichever files it fits. Users don't think in our file structure; they think in their business. The framework adapts.
