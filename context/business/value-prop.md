# Value Proposition

## Tagline

Stop re-explaining your business to AI. Onboard in 5 minutes; context grows as you ship.

## Pillars

1. **Instant first value (5-minute onboarding).** Seven 5W2H questions. Drafts 7 core files in one pass. One answer can populate multiple files — no ceremony.

2. **Zero-setup option for vibe coders.** "Build gradually" is the default: skip the upfront flow, the agent asks 1–2 questions only when it hits a context gap during normal work.

3. **Living document, not a spec.** The skill proposes updates as reality shifts. Drafts are cheap; only the user promotes to `active`. Context stays aligned without becoming a chore.

4. **No runtime, no lock-in.** Markdown files + YAML frontmatter + one Claude skill. MIT license. No infrastructure. Zero conflict with any existing repo setup.

5. **Selective loading prevents context rot.** Loading an entire business context file on every task degrades AI reasoning — models degrade measurably as input grows, losing focus on middle sections. RBC routes each task to 1–3 relevant files: the equivalent of RAG, without the infrastructure. Sharper responses, readable diffs, per-concept freshness tracking that a monolith can't provide.

6. **From developer tool to business partner.** Code is a commodity — AI writes it. The real constraint is business clarity: who you're building for, what problem actually matters, where the product should stop. Give the AI that context and it stops being an autocomplete engine. It advises on trade-offs, flags scope creep, pushes back when a feature doesn't fit the ICP. That's a business partner, not a code generator.

7. **One repo, every business output.** With MCPs, agents, skills, and modern IDEs, the same repo that ships your product can also generate social posts, sales pitches, customer service flows, media kits, and content briefs — all coherent, all on-brand, because they all draw from the same context. RBC turns the repo into a full business operating system.
