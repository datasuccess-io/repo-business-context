Usually codebases has a README.md file. It's more focused on humans, with much less detailed information and serves a different purpose.

IDEs like VS Code has been introducing features like `instructions.md`. This file is tottaly focused on AI coders and agents, giving instructions on how to behave, procedures, and line of work.

As a solo founder, who both think and develop new apps, I'm missing something to plug the business into the development process, given this modern environment and the context that AI should have while coding.

Apps are like a coin, with two sides: the business and the technology. Between them we have marketing, third-party providers, mental models, team, social media content, tasks, and much more - but they always serve one or another side of this coin. No need to stick too much into this analogy.

I really want the AI to know both, deeply.

- Business elements examples: business model, problem to solve, use cases, revenue streams, ideal persona, values and positioning, etc.
- Technology elements examples: front and back-end stacks, databases (tables, entities), tracking and observability, third-party providers, programming language, frameworks, libraries, etc.

This should be a set of documents, both human and AI readable. Always written in english, and well structured, and using Markdown markup language.

This document should stay updated to the project and must reflect it's reality. If there are examples, they reflect real use cases, entities, databases, tools, etc. If there are future features or ideas, they should be part of the overall vision, highly intentional, and connected to what is already planned within this set of documents.
